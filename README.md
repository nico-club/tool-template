# NICO Studio エリア2 — ツールテンプレート

クロードコード部「エリア2（開発ゾーン）」のひな形です。
**あなたの手元の AI（Claude Code / Codex / Cursor など何でも）で開発 → `main` に push するだけで、自動で本番URLに反映されます。**

## 仕組み（あなたがやること・やらないこと）

```
あなたの端末（好きなAIで開発） ── git push origin main ──▶ GitHub(nico-club)
                                                              │ 自動
                                                              ▼
                                                    Cloud Build → Cloud Run
                                                              ▼
                                                    公開URL（LINEに届く）
```

- ✅ やること：コードを書く（AIに書かせる）→ push。それだけ。
- ❌ やらないこと：サーバ設定・デプロイ作業・GCP操作（全部レール側が自動でやります）

## はじめかた

```bash
git clone https://github.com/nico-club/<あなたのrepo名>.git
cd <あなたのrepo名>
npm install
npm start        # http://localhost:8080 で動作確認
```

書き換えるのは基本3ファイル：

| ファイル | 役割 |
|---|---|
| `server.js` | 入力→処理→出力の本体（`/api/run`） |
| `public/index.html` | 画面 |
| `tool.json` | ツールの名刺（名前・説明・開発者名）。**最初に必ず自分の情報へ書き換える** |

リポジトリ直下に標準の `Dockerfile`（Node用）が入っています。**Node のまま使うなら触らなくてOK**。Python など別の言語で作りたい場合だけ、この `Dockerfile` を書き換えてください。レール側は「直下の `Dockerfile` をビルドする」だけなので、中身が何語でも同じ仕組みで公開されます。

## ルール（守れないと止めます）

1. **秘密情報（APIキー等）をコミットしない**。必要なら運営に申請（環境変数として設定を代行します）
2. ポートは `process.env.PORT` を使う（固定値にしない）
3. `/health` エンドポイントは消さない
4. 上限：ビルド10回/日・repo2本/人。CPU・メモリ・同時実行数はレール側で上限を固定しています（アプリ側で増やすことはできません）。マイニング・スパム・違法用途は即停止＋除名検討（利用規約参照）

## 困ったら

- push したのに反映されない → ビルド失敗の可能性。LINE の通知（または運営）へ
- 質問は部の LINE へ。**エラーメッセージを貼るのが最速**
